    Featured Projects

    Guitar Tracker

    This full-stack web application uses Rails and SQL on the server with JavaScript, jQuery and AJAX on the front-end. The site helps guitarists keep track of the configurations of their guitars. Bootstrap modals are used for forms, and Handlebars are used to dynamically render content on the page. When a user adds, deletes, or updates a guitar, their catalog immediately reflects the changes they made.

    Using handlebars was not a requirement, but I love how it improves the user experience. Without it, a user would need to refresh the page or click on a button to see the changes they've made. At first I didn't want to use them because I barely understood how they worked, but once I met the requirements for the project I wanted to add features that improved the user experience. This is also why I added a delete confirmation so users wouldn't accidentally delete a guitar. This was something I thought would be simple but ended up being a challenge! I used data-ids on the html elements to target the right guitar for actions, since I was using modals I had to connect the modal with whatever guitar the user was trying to target. Although I could have spent my time making the site more aesthetically pleasing, I'm glad I went for these challenges that required me to learn new technologies and techniques.

    MechanicMate

    I built this app as my capstone project for General Assembly. The API is built with Ruby on Rails, using PostgreSQL to interact with the SQL database. The front-end is built with Ember.js, a framework that allows single-page applications to route to numerous URLs.

    The development cycle was 1 week, which made this project a great challenge, as it was my first experience using a front-end framework. On top of that, we'd spent only a few days learning Ember in class. Continuing to learn about Ember while developing this website was crucial to my success. Completing this project made me confident that I can learn new tech on the job, and learn the fundamentals of a framework in the span of a week.

    As an extra challenge, I decided to ask my sister, a graphic designer, for help with styling the site. It's one thing to be able to write your own styling code, but I wanted to see if I could be given a design and then write code that puts that design on the web. I know there are many development jobs where that's the workflow. She designed the splash page, and I used that design to inform aesthetic decisions for the rest of the site. Making the website match that design was more difficult than I thought it would be, and it was a great learning lesson. The result of having a website with a unique, cohesive style was worth all of the extra effort!

    Guitar Tacker 2.0

    This is an in-progress, improved version of Guitar Tracker that I'm rebuilding from the ground up, using React on the front-end. The SQL database and Rails API has been changed to allow for guitar histories. Now, when users update their information (such as when they changed their strings) the old value isn't overwritten - it's saved in the guitar's history. What sounds like a simple change required a complete restructuring of the database, which is why I didn't do this for version 1, which had to be completed in under a week. As development is in its early stages, the site isn't live, but the github repos are public.

    The Glades Website
